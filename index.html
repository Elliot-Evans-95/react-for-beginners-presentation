Why React Exists

In 2013 Websites were being built using html templates. HTML template dictate how you should abstract your website. So in 2013 Facebook created a javascript library called React. React breaks that mould by breaking down user interface abstraction into components. Components can be used throughout the site which makes it easier to create large scale applications. Components are make up of your view logic and your markup, by having both in one place it makes applications easier to maintain and extend.

Why am I only comparing react and angular?
Angular
Platform:
Angular is a platform, a large framework with a battery included approach. So Angular is almost a MVC framework for the frontend, compared to react which is just the View
Data binding:
Angular can be used as either one-way data binding or two-way data binding. Explain one way data binding, explain two-way data binding.
“Angular Way”:
Angular has its own recommendation in how you should write your code in a angular app. Its known the “Angular Way”. React does not care about how you write it, only in a few small cases (arrays not objects)
Typescript:
Angular is not written in javascript but in Typescript. Typescript is made by microsoft to add types in javascript. It pushes the OO way of programming. React is ES6+ and takes a more functional way of programming.
HTML:
Angular components are split into the Typescript file, the Style file and then the HTML file. The Style and HTML are referenced by the Typescript file. This makes it easier to switch between typical website making and that of components. React has all logic and view in one file and also the style gets referenced.
Why is Angular Slower
Due to a startup time but its faster once the loading has been finished.
